# Journal

Chronological session log — the narrative memory of the project. One file per day
(`YYYY-MM-DD.md`); multiple sessions in a day are `## HH:MM` sections within it.

- **Append-only.** Never rewrite history; correct later entries instead.
- Each session adds an entry at end-of-session using the template in `../HANDOFF.md`.
- This is the *story* ("what happened, when, why, what's next"); the current *facts*
  live in `../../project-state.md` and `../../todo.md`. When in doubt about the present,
  trust project-state; when in doubt about how we got here, read the journal.

To resume, read the **latest** entry first (it ends with a "Next step").
