Πηγή: «CDA Documentation v.1.0.1.8» (epSOS **ePrescription** + **eDispensation**).
Στόχος: ποια πεδία του CDA αξιοποιούμε ήδη, ποια **πρέπει** να αξιοποιήσουμε, και πού
στο project κάνουν τα δεδομένα πιο σωστά/δομημένα.

Mapping → canonical: `hdika_client.py::_map_full`. Persist: `engine.py`.

## C. Δομικές βελτιώσεις
1. **Per-line `outstanding` (1.4.19)** → η αυθεντική πηγή για «ανεκτέλεστα» αντί
   inference από statusCode· λύνει και το «τι δόθηκε σε αυτή την εκτέλεση».
2. **Generic classification** (1.9.6.1 + 1.4.15 + 1.9.6.2) → ενιαίο `generic`/
   `original` σε επίπεδο είδους → profitability generic-mix + σωστά ΚΥΥΑΠ caps.
3. **execution_case (1.1.18)** → ακριβές status της εκτέλεσης.
4. **Prescription difference totals (1.2.1.x)** → cross-check των υπολογισμένων ποσών.

B[1.10.4] — αποθηκεύονται structured στο item.details / execution.details.
